            # Memory

            # Objective

            For the final project you will be demonstrating your understanding JavaScript, CSS, and web animation to create a Memory game.

            # Game Play

            When the page loads the player will be presented with a set of difficulty options. Once the player has selected a difficulty, the cards will appear and the game will begin.

            The player will click on any card to show the card's pattern. The card's pattern will remain shown until a second card is clicked. If the second card's pattern matches the first, the game will indicate that a match has been found. If the second card's pattern does not match the first, the both patterns will be hidden. The player will continue selecting cards until all of the matches have been found.

            Once all matches have been found the player with an end of game sequence. Then the player should be presented with the option to play the game again.

            # Requirements

            The following requirement must be met in order to complete the assignment successfully:

            1. When the page loads, the screen must display a set of difficulty levels.
            2. There must be a minimum of 3 difficulty levels. NOTE: Difficulty level can be created simply by adding or removing the number of cards on the board. There should be at least 4 cards for each difficulty level.
            3. Selecting a difficulty level should display a grid of cards.
            4. The cards must be laid out in a user-friendly manner using Flexbox.
            5. The cards should be created and added to the page dynamically using JavaScript.
            6. Each card should have a pattern (image, shape, symbol, etc) that is hidden from the player.
            7. Clicking on a card should show the pattern. Showing the pattern MUST be animated.
            8. If the second card matches the first, there must be some indication that there was a match. For example, the matched cards stay open, or fade out or disappear all together.
            9. If the second card does not match the first, both cards should hide their patterns. Hiding the pattern MUST be animated.
            10. When all the matches have been found, an ending sequence must be initiated. The ending sequence must be animated, using @keyframes rules and the animation property.
            11. At the end of the sequence the player must be presented with the option to play again.
